# Error codes

The stable operator-facing registry is:

- `SIGNATURE_INVALID`, `TIMESTAMP_EXPIRED`, `SCHEMA_UNSUPPORTED`, `PAYLOAD_INVALID`, `PAYLOAD_TOO_LARGE`, `PAYLOAD_HASH_MISMATCH`
- `LOCK_LOST`, `WORKER_INTERRUPTED_BEFORE_DISPATCH`, `WORKER_INTERRUPTED_IN_FLIGHT`
- `ACTION_SCHEDULER_NOT_READY`, `ACTION_SCHEDULER_VERSION_UNSUPPORTED`, `ACTION_SCHEDULER_PREFLIGHT_REQUIRED`, `ACTION_SCHEDULE_FAILED`, `ACTION_ID_AMBIGUOUS`, `ACTION_ID_MISMATCH`
- `CHECKPOINT_CONFLICT`, `RESUME_PHASE_CONFLICT`, `ADAPTER_RESPONSE_INVALID`, `ATTEMPTS_EXHAUSTED`
- `HUBSPOT_AUTH`, `HUBSPOT_SCHEMA`, `HUBSPOT_RETRYABLE`
- `SLACK_AUTH`, `SLACK_TERMINAL`, `SLACK_RETRYABLE_BEFORE_POST`, `SLACK_OUTCOME_UNKNOWN`
- `PAYMENT_FACT_MISSING`, `CANCELLATION_FACT_MISSING`, `CANCELLATION_FACT_INVALID`, `REFUND_FACT_MISSING`, `RECONCILIATION_FACT_FAILURE`
- `OPERATOR_WAIT_REQUIRES_RESOLVE_OUTCOME`, `OPERATOR_WAIT_EPOCH_CONFLICT`, `RESOLUTION_NOT_ALLOWED_FOR_REASON`, `ACTION_CONFLICT`

Errors stored in WordPress are sanitized and bounded. Authentication headers, shared secrets, tokens, full webhook URLs, and raw personal fields are never valid error content.
