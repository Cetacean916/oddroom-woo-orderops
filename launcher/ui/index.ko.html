패키지 상태를 확인하고 있습니다.

01

런타임과 모드

로컬 런타임을 시작하고 DEMO_MODE 또는 CONNECTED_MODE를 선택합니다. 모드를 바꿔도 같은 주문과 설정을 사용합니다.

02

상점과 관리자

준비 검사를 통과한 WooCommerce 상점과 관리자 화면을 엽니다. 관리자 계정은 이 패키지 안에서 생성됩니다.

03

데모 상태와 복구

다음 합성 주문의 정상·재시도·실패·검토 상태를 선택합니다. 처리 결과와 복구 흐름은 관리자 화면에서 확인합니다.

04

런타임 중지

컨테이너를 중지하고 주문·설정·볼륨은 보존합니다. 다시 시작하면 같은 상태를 이어갑니다.

DEMO DATA CONTROL

데모 상태와 초기화

선택한 상태와 초기화 작업은 패키지가 소유한 합성 데모 데이터에만 적용됩니다. 외부 CRM·Slack·결제 서비스는 호출하지 않습니다.

적용 후 상점에서 새 합성 주문을 만들면 선택한 결과가 작업자 처리 경로에 반영됩니다.

정확히 RESET PF07 DEMO를 입력해야 합니다. 관리자 계정·카탈로그·런타임 볼륨은 보존됩니다.

PACKAGE OPERATIONS

백업·복원·이관

이 영역의 작업은 PF07 Compose 프로젝트와 볼륨만 대상으로 합니다. 백업과 증거 파일은 추출 폴더 밖에 만들며 토큰과 관리자 비밀번호는 진단에서 제외합니다.

암호화 백업

세 볼륨과 로컬 상태를 일시 정지한 뒤 AES-256-CTR과 HMAC-SHA256으로 보호합니다. passphrase는 저장하지 않으므로 별도로 보관해야 합니다.

백업 복원

현재 상태가 있으면 복원 전 백업을 먼저 만들고, 기존 런타임을 중지한 뒤 PF07 볼륨만 교체합니다.

1.0.1 상태 이관

두 패키지의 무결성과 build ID를 확인하고 기존 런타임을 중지한 뒤 `SHOP_INSTANCE_ID`와 세 볼륨을 이 패키지에 연결합니다.

HTTPS 터널

상점과 WordPress 관리자 경로만 공개합니다. n8n·DB·Docker API·로그·증거는 공개하지 않으며 터널을 중지해도 로컬 상점은 계속 동작합니다.

패키지 제거

필요한 암호화 백업을 먼저 만드세요. 다른 Compose 프로젝트·볼륨·파일은 건드리지 않습니다.
