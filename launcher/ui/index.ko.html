DETERMINISTIC DEMO CONTROL

상태 재현과 범위 제한 초기화

이 제어는 패키지의 합성 데모 데이터에만 적용됩니다. 외부 CRM·Slack·결제 서비스는 호출하지 않습니다.

적용 후 상점에서 새 합성 주문을 만들면 선택한 상태가 실제 작업자 경로에 반영됩니다.

정확히 RESET PF07 DEMO를 입력해야 합니다. 관리자 계정·카탈로그·런타임 볼륨은 보존됩니다.

PACKAGE OPERATIONS

진단부터 안전한 이관까지.

모든 작업은 이 패키지의 Compose 프로젝트와 볼륨만 대상으로 합니다. 증거와 암호화 백업은 추출 폴더 밖에 생성되며 토큰·관리자 비밀번호는 진단에 포함되지 않습니다.

인증된 암호화 백업

세 볼륨과 로컬 상태를 잠시 quiesce한 뒤 AES-256-CTR + HMAC-SHA256 Encrypt-then-MAC으로 보호합니다. passphrase는 저장되지 않으며 분실 시 복구할 수 없습니다.

한 writer로 복원

현재 상태가 있으면 같은 passphrase의 외부 pre-restore 백업을 먼저 만들고, 기존 writer를 중지한 뒤 정확한 PF07 볼륨만 교체합니다.

통제된 사전 배포 업데이트

새 아카이브를 별도 폴더에 풀어 실행합니다. 무결성과 build ID를 확인하고 기존 런타임을 quiesce한 뒤 `SHOP_INSTANCE_ID`와 세 볼륨을 하나의 successor writer에 연결합니다.

옵션 HTTPS 터널

상점과 WordPress 인증이 유지되는 관리 경로만 allowlist합니다. n8n·DB·Docker API·로그·증거는 공개하지 않으며, 터널 실패와 중지 후에도 로컬 상점은 계속 동작합니다.

패키지 범위 제거

필요한 암호화 백업을 먼저 만드세요. 다른 Compose 프로젝트·볼륨·파일은 건드리지 않습니다.
